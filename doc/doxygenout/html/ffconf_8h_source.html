94#define FF_USE_LFN 3
95#define FF_MAX_LFN 255
96/* The FF_USE_LFN switches the support for LFN (long file name).
97/
98/ 0: Disable LFN. FF_MAX_LFN has no effect.
99/ 1: Enable LFN with static working buffer on the BSS. Always NOT thread-safe.
100/ 2: Enable LFN with dynamic working buffer on the STACK.
101/ 3: Enable LFN with dynamic working buffer on the HEAP.
102/
103/ To enable the LFN, ffunicode.c needs to be added to the project. The LFN function
104/ requiers certain internal working buffer occupies (FF_MAX_LFN + 1) * 2 bytes and
105/ additional (FF_MAX_LFN + 44) / 15 * 32 bytes when exFAT is enabled.
106/ The FF_MAX_LFN defines size of the working buffer in UTF-16 code unit and it can
107/ be in range of 12 to 255. It is recommended to be set it 255 to fully support LFN
108/ specification.
109/ When use stack for the working buffer, take care on stack overflow. When use heap
110/ memory for the working buffer, memory management functions, ff_memalloc() and
111/ ff_memfree() exemplified in ffsystem.c, need to be added to the project. */
